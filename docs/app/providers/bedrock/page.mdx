# AWS Bedrock

Use AWS Bedrock for batch inference with Claude models.

import { Callout } from 'nextra/components'

## Prerequisites

- AWS account with Bedrock access
- S3 bucket for batch I/O
- IAM role with required permissions

## Configuration

```bash
# .env
DEFAULT_PROVIDER=bedrock
AWS_REGION=us-east-1
AWS_PROFILE=your-profile
BEDROCK_S3_BUCKET=your-convoy-bucket
BEDROCK_ROLE_ARN=arn:aws:iam::123456789012:role/BedrockBatchRole
BEDROCK_S3_INPUT_PREFIX=batch-inputs
BEDROCK_S3_OUTPUT_PREFIX=batch-outputs
```

## S3 Bucket Setup

Create an S3 bucket for batch input/output files:

```bash
aws s3 mb s3://your-convoy-bucket --region us-east-1
```

## IAM Role

Create an IAM role with these permissions:

### Trust Policy

```json
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Effect": "Allow",
      "Principal": {
        "Service": "bedrock.amazonaws.com"
      },
      "Action": "sts:AssumeRole"
    }
  ]
}
```

### Permissions Policy

```json
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Effect": "Allow",
      "Action": [
        "s3:GetObject",
        "s3:PutObject",
        "s3:ListBucket"
      ],
      "Resource": [
        "arn:aws:s3:::your-convoy-bucket",
        "arn:aws:s3:::your-convoy-bucket/*"
      ]
    }
  ]
}
```

### Worker Permissions

The Convoy worker needs these permissions:

```json
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Effect": "Allow",
      "Action": [
        "bedrock:CreateModelInvocationJob",
        "bedrock:GetModelInvocationJob",
        "bedrock:ListModelInvocationJobs",
        "bedrock:StopModelInvocationJob"
      ],
      "Resource": "*"
    },
    {
      "Effect": "Allow",
      "Action": [
        "s3:GetObject",
        "s3:PutObject",
        "s3:ListBucket"
      ],
      "Resource": [
        "arn:aws:s3:::your-convoy-bucket",
        "arn:aws:s3:::your-convoy-bucket/*"
      ]
    },
    {
      "Effect": "Allow",
      "Action": "iam:PassRole",
      "Resource": "arn:aws:iam::123456789012:role/BedrockBatchRole"
    }
  ]
}
```

## Supported Models

Convoy uses provider-agnostic model IDs. When using Bedrock, these are automatically translated to Bedrock model identifiers.

<Callout type="info">
  See [Supported Models](/concepts/models) for the full list of models and regional availability.
</Callout>

| Model ID | Bedrock Model ID |
|----------|------------------|
| `claude-3-haiku` | `anthropic.claude-3-haiku-20240307-v1:0` |
| `claude-3-sonnet` | `anthropic.claude-3-sonnet-20240229-v1:0` |
| `claude-3-opus` | `anthropic.claude-3-opus-20240229-v1:0` |
| `claude-3.5-haiku` | `anthropic.claude-3-5-haiku-20241022-v1:0` |
| `claude-3.5-sonnet` | `anthropic.claude-3-5-sonnet-20241022-v2:0` |
| `claude-sonnet-4` | `anthropic.claude-sonnet-4-20250514-v1:0` |
| `claude-opus-4` | `anthropic.claude-opus-4-20250514-v1:0` |

### Regional Availability

Not all models are available in all AWS regions. Convoy validates model availability based on your configured `AWS_REGION`. If a model is not available in your region, you'll receive a `400 Bad Request` error with the list of available regions.

## Example Request

Use the provider-agnostic model ID in your requests:

```bash
curl -X POST http://localhost:8000/cargo/load \
  -H "Content-Type: application/json" \
  -H "X-API-Key: convoy_sk_your_key_here" \
  -d '{
    "params": {
      "model": "claude-3-haiku",
      "max_tokens": 100,
      "messages": [{"role": "user", "content": "Hello"}]
    },
    "callback_url": "https://example.com/callback"
  }'
```

## How It Works

1. You submit a request with a provider-agnostic model ID (e.g., `claude-3-haiku`)
2. Convoy validates the model and checks regional availability
3. The model ID is translated to the Bedrock model identifier
4. Convoy collects requests and writes them to S3 as JSONL
5. Creates a Bedrock batch inference job
6. Polls job status until complete
7. Reads results from S3 output location
8. Delivers results via callbacks
